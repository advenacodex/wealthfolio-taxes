# Agent Workflow Rules

Follow this sequence strictly:

1. Read BACKLOG.md
2. Select ONLY the FIRST task in "# Pending"
3. Move it to "# In Progress"
4. Implement the task
5. Validate the implementation
6. Move the task to "# Done"
7. Append a short technical summary to CHANGELOG.md. Use grouping by date
8. Stop after one task

Rules:
- Never work on multiple tasks
- Never reorder tasks
- Never create new tasks unless required
- Preserve task IDs
- Keep changes minimal and reversible
